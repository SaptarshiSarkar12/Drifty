# Contributing
Welcome to the contributing guide for the project. This guide will help you get started with contributing to the project.

## Labels
The project uses labels to categorize issues and pull requests. The following labels are used:
- `bug` : The issue or pull request is a bug.
- `enhancement`: The issue or pull request is an enhancement.
- `help wanted`: The issue or pull request needs help.
- `good first issue`: The issue or pull request is a good first issue. This means that the issue or pull request is easy to fix or implement by someone who is new to the project.
- `documentation`: The issue or pull request is related to documentation.
- `App`: The issue or pull request is related to the app.
- `Website`: The issue or pull request is related to the website.
- `assets`: The issue or pull request is related to the assets like the logo, app icon, etc.
- `dependencies`: The issue or pull request is related to the dependencies.
- `maven`: The issue or pull request is related to the Maven dependencies.
- `npm`: The issue or pull request is related to the npm dependencies.
